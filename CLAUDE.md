# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Purpose

Builds a canonical, publishable knowledge base of Hong Kong addresses, estates,
streets, and POIs for RAG and LLM use. The gov.hk **ALS** (Address Lookup
Service) GeoJSON is the primary source; **OpenStreetMap** (via Overpass /
Nominatim) supplies enrichment. See [docs/data-dictionary.md](docs/data-dictionary.md)
for the published schema and [README.md](README.md) for the build quickstart.

## Environment & commands

Python with a local virtualenv. Activate before doing anything:

```bash
source .venv/bin/activate          # deps: pip install -r requirements.txt
```

Tests import scripts as `from bin.X import ...`, so **always run pytest from the
repo root** (no pytest config file exists):

```bash
python -m pytest tests/                                   # full suite
python -m pytest tests/test_fix_district_names.py          # one file
python -m pytest tests/test_fix_district_names.py::test_map_district_chinese  # one test
```

There is no build or lint step.

## Pipeline architecture

The dataset is produced by an ordered chain of `bin/` scripts. Each stage reads
the previous stage's output file from `data/` and writes the next — understand
the data handoffs before changing any single script:

1. `download_geojson.py` → ALS GeoJSON (per-district files), ETag-aware.
2. `geojson_to_canonical.py` → `data/als_canonical.csv` (~159k rows). Flattens
   the nested ALS structured-address JSON into flat bilingual `*_chi`/`*_eng`
   columns; dedupes on the ALS `id` (CSU / GeoAddress key).
3. `export_places_os.py` → `data/address_data.json`. Overpass dump of named OSM
   elements for HK. `build_broad_query` (all named elements) is the current path
   but is resource-heavy; `build_overpass_query` is the narrower per-tag-type
   alternative.
4. `fix_district_names.py` → `data/als_canonical_fixed.csv`. The key enrichment
   stage (see below).

Independent, agent-driven track (not part of the CSV pipeline):
`suggest_entities.py` scans ALS for missing entities → `entity_create.py`
scaffolds an `entities/*.md` file → `enrich_entity.py` adds a `geometry_ref` via
Nominatim. `entities/` is the curated Markdown+YAML-frontmatter knowledge base;
`fetch_nominatim.py` / `search_nominatim.py` are its cached OSM lookup helpers.
`spatial_join_processor.py` is a standalone OSM building/address/admin spatial-join
utility, not wired into the main chain.

## The central problem: administrative vs colloquial districts

This is the repo's reason to exist. ALS tags every address only with one of
Hong Kong's 18 **administrative** districts (e.g. `油尖旺區`), which nobody uses
in speech. `fix_district_names.py` adds a `sub_district` with the **colloquial
locality** people actually say (`旺角`, `尖沙咀`, `油麻地`):

- `DISTRICT_MAP` only *normalises* the 18 admin district spellings — it does not
  solve the colloquial problem.
- `process_df` does, in two stages: (1) `_nearest_localities` finds the nearest
  OSM locality label (`place` = suburb/quarter/neighbourhood/village/town/region)
  within `max_m` (default **1500 m**) using `geopandas.sjoin_nearest` projected to
  HK80 (EPSG:2326) — distance-first, vectorised; (2) a village-name fallback for
  rows with no label in range. `sub_district_dist_m` records match distance (null
  ⇒ came from the fallback) and is the confidence signal.
- `--inject-into-address` (opt-in, default off) additionally splices the locality
  into `address_chi`/`address_eng` next to the district token.

When tuning matching quality, the levers are: the `place` types accepted in
`DEFAULT_PLACE_TYPES`, and `max_m` (smaller = stricter, fewer cross-boundary
mis-assignments but more nulls).

## Gotchas

- **`data/` and `*.ipynb` are gitignored.** The large datasets and `data.ipynb`
  (an exploratory scratchpad mirroring `fix_district_names.py` logic) are not
  tracked. Don't assume a fresh checkout has any `data/` files — they must be
  regenerated by the pipeline.
- **Coordinates use two CRS:** `lat`/`lon` are WGS84 (EPSG:4326); `easting`/
  `northing` are HK80 grid metres (EPSG:2326). Distance work must project to 2326.
- **Tests and code have drifted historically** — when editing a `bin/` script,
  check its test still reflects the real function signatures.
